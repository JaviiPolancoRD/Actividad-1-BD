Actividad-1 Conceptos Basicos


1. ¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, etc.) de un atributo o variable. Por ejemplo, el número 25 podría ser un dato que representa la edad de una persona.

2. ¿Qué es información?

La información es el conjunto de datos organizados y procesados de manera que tengan valor y significado. Por ejemplo, si tienes una lista de edades de personas y calculas el promedio de esas edades, el resultado sería información derivada de los datos originales.

3. ¿Diferencia entre dato e información?

Un dato es la representación simbólica de una variable, mientras que la información es el resultado del procesamiento y organización de datos que tiene valor y significado para los usuarios.

4. ¿Tipos de datos?

Los tipos de datos varían según el contexto, pero comúnmente incluyen datos numéricos (enteros, decimales), datos de texto (cadenas), datos booleanos (verdadero/falso), entre otros según el lenguaje de programación o sistema de gestión de bases de datos que se esté utilizando.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo de datos para una base de datos. Este modelo define la estructura lógica de los datos y las relaciones entre ellos.

6. ¿Tipos de bases de Datos, según:

- Variabilidad: Bases de datos estáticas (no cambian con el tiempo) y bases de datos dinámicas (se actualizan frecuentemente). - Contenido: Bases de datos textuales (almacenan principalmente texto) y bases de datos multimedia (almacenan datos como imágenes, audio, video). - Modelo de base de datos: Bases de datos relacionales (estructuradas en tablas relacionadas), bases de datos NoSQL (estructuras flexibles que no requieren un esquema fijo), entre otros.

7. ¿Qué es el modelo relacional?

El modelo relacional es un modelo de datos para la gestión de bases de datos que organiza los datos en tablas relacionadas entre sí mediante claves.

8. ¿Describa los elementos del Diagrama entidad Relacional?

- Los elementos principales son: - Entidades (objetos del mundo real, como Personas, Productos). - Atributos (propiedades de las entidades, como Nombre, Precio). - Relaciones (conexiones entre entidades, como Vende, Posee). - Claves Primarias y Foráneas (identificadores únicos y relaciones entre tablas).

9. ¿Qué es una entidad?

Una entidad representa un objeto del mundo real (como una persona, un libro, etc.) que puede ser identificado y distinguible de otros objetos.

10. ¿Qué es un atributo y sus tipos?

Un atributo es una propiedad o característica que describe una entidad. Puede ser de tipos como numéricos, alfanuméricos, fechas, booleanos, entre otros.

11. ¿Qué es una relación?

Una relación es una asociación entre dos o más entidades que representa la conexión o interacción entre ellas en la base de datos.

12. ¿Qué son las llaves primarias, foránea y sus características?

La llave primaria es un atributo o conjunto de atributos que identifica de manera única a cada registro en una tabla. La llave foránea es un atributo o conjunto de atributos que establece una relación entre dos tablas, vinculando la llave foránea de una tabla con la llave primaria de otra.

13. ¿Qué son las reglas del Negocio?

Las reglas del negocio son restricciones y políticas que deben cumplirse en un entorno de negocio específico. En el contexto de las bases de datos, estas reglas pueden influir en el diseño del esquema de la base de datos y en las operaciones permitidas sobre los datos.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad en el contexto de las bases de datos describe las relaciones entre las entidades, especificando cuántas instancias de una entidad pueden estar asociadas con cuántas instancias de otra entidad. Puede ser uno a uno, uno a muchos, muchos a uno o muchos a muchos.
